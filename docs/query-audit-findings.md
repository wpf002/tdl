# Query Audit Findings & Remediation Plan

**Date:** 2026-06-03
**Audit artifact:** `exports/audit_results.json` (committed)
**Audit cost:** $48.40 (821 rules)

## 1. What was done

Every rule in the library (821) was semantically audited by the 10 per-language
SIEM specialist agents (`tools/agents/`, Sonnet 4.5) plus a cross-cutting
generalist (Haiku 4.5), via `tools/audit_rules_semantic.py`. Each specialist
graded its query 0–100 for syntax validity, correct fields/tables, and fidelity
to the rule's `pseudo_logic`, and listed specific issues.

## 3. Root cause — confirmed

The queries were generated by `tools/regen_queries.py`, a **template renderer**.
It classifies each rule into one of ~6 families (auth / cloud / network /
process / registry / file) from its `data_sources`/`pseudo_logic`, then renders
a generic per-family shell with a single substituted filter (one event ID, a
port list, an operation name, etc.) via `tools/gen/query_templates.render()`.

This is structurally incapable of expressing the rule-specific logic the
`pseudo_logic` describes:

- **Thresholds dropped** — "7 events", "30 firewall events", "10 within 3 min"
  all render as `| where count >= 1`.
- **Time windows dropped** — no `bin _time` / `ago()` windows.
- **Multi-condition logic dropped** — encryption types, failure codes,
  application names, allow-list exclusions (`NOT IN`) omitted; one rule's
  allowlist was even **inverted** (`IN` instead of `NOT IN`).
- **Partial event-ID sets** — `[673, 4769]` → only `4769`; correlation
  ("Followed By") flattened to a flat `IN (...)`.
- **Wrong log source family** — O365/Exchange rules render AWS/Azure/GCP
  CloudTrail queries; a Kubernetes rule queried cloud-audit logs.
- **Per-dialect field/syntax errors** — `event_id` vs `EventCode`, invalid
  inline LEQL `logset=`, wrong YARA-L/XQL event-type enums, placeholder
  regexes like `process_name="*.+*"`.

**Verification:** `python3 tools/regen_queries.py --rule TDL-CA-000019 --dry-run`
reproduces the current broken Kerberoasting query verbatim — confirming the
template engine is the source, and that re-running it cannot fix the mismatch.

## 4. Remediation — validated by pilot

Regenerate each rule's queries **from its `pseudo_logic`** using the specialist
agents' `generate_query` path (`AgentOrchestrator.generate_all_queries`), which
reads the full logic and produces faithful, dialect-correct queries.

Pilot (2 rules, regenerate → re-validate):

| Rule | Old → New score | Fix |
|---|---|---|
| `TDL-CA-000019` Kerberoasting | **9 → 69** | EventCode 673+4769, `Failure_Code=0x0`, enc types 0x17/0x18, ticket options, krbtgt/machine exclusions, 1-min binning |
| `TDL-DE-000027` Set-InboxRule | **11 → 65** | Correct `index=o365 …Operation="Set-InboxRule"`; forward/redirect target + external-domain check |

LLM regen lifts scores from "broken" (~10) to usable (~65–69). Template regen
cannot.

   `AgentOrchestrator.generate_all_queries` for generation and the YAML literal

   Add `--resume`, `--max-cost`, and credit-aware abort like
   `audit_rules_semantic.py`.

   resumable. Est. **$50–70**, ~2–3 hr. Will span credit refills.

   the score lift before any merge to `main`.
